SECTION 1 — WHAT SIREN.NG IS
================================================================================

Siren.ng is a community emergency coordination platform for Lagos, Nigeria.

The core loop:
  1. Someone sees an emergency and sends a WhatsApp message or uses the website
  2. Claude AI reads the report and verifies it within 90 seconds
  3. If verified, the nearest community responder and clinic are notified along with members around that location or who subscribed to monitor that location.
  4. The reporter gets a live tracking link
  5. The community can vouch for, donate to, and contribute resources
  6. When resolved, everyone sees the loop close

Two entry points, one API:
  WhatsApp   Zero friction, no account (Phase 2)
  Web        Richer experience, same API underneath (Phase 1 complete)

Seven accepted incident types only:
  FIRE  FLOOD  COLLAPSE  RTA  EXPLOSION  DROWNING  HAZARD
  Everything else is rejected by the AI with an explanation.

SECTION 2 — TECHNOLOGY STACK
================================================================================

BACKEND
--------------------------------------------------------------------------------
  Python 3.13                            Language
  Django 5.1                             Web framework and ORM
  Django REST Framework 3.16             API layer — serializers, views, permissions
  djangorestframework-simplejwt 5.5      JWT auth for agency/admin users
  django-cors-headers                    Allows browser fetch() calls to the API
  Django Channels 4.3                    WebSocket support for real-time updates
  channels-redis 4.3                     Redis backend for the WebSocket channel layer
  Celery 5.6                             Background task queue
  Anthropic SDK                          Claude AI calls for incident verification
  Twilio 9.10                            WhatsApp Business API (Phase 2)
  Cloudinary                             Media/image storage for incident photos
  psycopg2-binary                        PostgreSQL driver
  dj-database-url                        Parses DATABASE_URL from environment
  python-decouple                        Reads .env variables cleanly via config()
  whitenoise                             Serves static files without a separate CDN

FRONTEND (current — Phase 1, no build step needed)
--------------------------------------------------------------------------------
  Django templates             Server-rendered HTML pages
  Tailwind CSS via CDN         All styling — loaded from https://cdn.tailwindcss.com
  Alpine.js via CDN            Reactive data binding and DOM updates
  Vanilla fetch()              All API calls from the browser
  Native WebSocket API         Live incident tracking on /track/<id>/

FRONTEND (planned — Phase 6)
--------------------------------------------------------------------------------
  React 19 + Vite 6 + TypeScript 5       Main framework
  react-leaflet + leaflet                Interactive map, OpenStreetMap tiles (free, no API key)
  recharts                               Analytics charts
  axios + @tanstack/react-query          Data fetching and caching
  zustand                                Global state management
  react-router-dom 6                     Client-side routing

DATABASE 
--------------------------------------------------------------------------------
  Supabase (PostgreSQL)    Hosted Postgres. Use PLAIN engine (django.db.backends.postgresql)
                           NEVER use django.contrib.gis.db — no PostGIS, no GeoDjango, ever.
  Connection string:       Direct Connection URI from Supabase Settings > Database
  NOTE: If you get an IPv4 error with the Direct URI, use the Pooler URL (port 6543).
        The Pooler works fine for development. Use the Direct URI in production.

INFRASTRUCTURE
--------------------------------------------------------------------------------
  Redis (Railway)    Celery task broker + Django Channels layer
  Railway            Django deployment — connect your GitHub repo
  Cloudinary         Incident photo storage
  Paystack           Donations and automatic fund splitting (Phase 4)
  OpenStreetMap      Free map tiles — no API key needed (Phase 6)

SECTION 4 — ENVIRONMENT VARIABLES (.env)
================================================================================

Create a file named .env in the project root. Never commit it to git.

  SECRET_KEY=any-random-string-at-least-50-characters-long
  DEBUG=True
  ALLOWED_HOSTS=localhost,127.0.0.1
  CELERY_ALWAYS_EAGER=True

  # Supabase — Direct Connection URI from Settings > Database
  DATABASE_URL=postgresql://postgres:[PASSWORD]@db.[REF].supabase.co:5432/postgres

  # Railway Redis — from Redis service Variables tab
  REDIS_URL=redis://default:[PASSWORD]@[HOST]:[PORT]

  # Anthropic Claude
  ANTHROPIC_API_KEY=sk-ant-...
  ANTHROPIC_MODEL=claude-sonnet-4-6

  # Twilio WhatsApp (Phase 2)
  TWILIO_ACCOUNT_SID=AC...
  TWILIO_AUTH_TOKEN=...
  TWILIO_WHATSAPP_NUMBER=whatsapp:+234XXXXXXXXXX

  # Paystack (Phase 4)
  PAYSTACK_SECRET_KEY=sk_live_...
  PAYSTACK_PUBLIC_KEY=pk_live_...
  SIREN_PAYSTACK_SUBACCOUNT=ACCT_...

  # Cloudinary
  CLOUDINARY_CLOUD_NAME=...
  CLOUDINARY_API_KEY=...
  CLOUDINARY_API_SECRET=...

KEY NOTES:
  CELERY_ALWAYS_EAGER=True  Tasks run synchronously inside the same Django
                            request. POST /api/ingest/web/ blocks ~2-5 seconds
                            while Claude processes the report. Expected in dev.
                            Set False in production and run a Celery worker.

  No Redis in dev           WebSocket broadcasts silently fail (wrapped in
                            try/except). Status saves are unaffected.

SECTION 5 — HOW TO RUN LOCALLY
================================================================================

CRITICAL: Always use the venv Python — not the system Python.

  CORRECT:  .venv/Scripts/python.exe manage.py runserver
  WRONG:    python manage.py runserver

The system Python does not have the project packages installed.

FIRST TIME SETUP:
  .venv/Scripts/python.exe manage.py migrate
  .venv/Scripts/python.exe manage.py createsuperuser
  .venv/Scripts/python.exe manage.py runserver

AFTER MODEL CHANGES:
  .venv/Scripts/python.exe manage.py makemigrations
  .venv/Scripts/python.exe manage.py migrate

INSTALL A NEW PACKAGE:
  .venv/Scripts/python.exe -m pip install <package-name>

CHECK EVERYTHING IS WIRED UP:
  .venv/Scripts/python.exe manage.py check

PAGES (no build step — just run the server):
  http://localhost:8000/          Home page
  http://localhost:8000/report/   Report an incident
  http://localhost:8000/feed/     Browse incidents
  http://localhost:8000/track/    Enter a tracking ID
  http://localhost:8000/admin/    Django admin panel

RAW API:
  http://localhost:8000/api/incidents/
  http://localhost:8000/api/incidents/active/
  http://localhost:8000/api/ingest/web/
