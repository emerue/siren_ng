SECTION 1 — WHAT WE ARE BUILDING
================================================================================

Siren.ng is a community emergency coordination platform for Lagos, Nigeria.

The problem: 53% of emergency calls in Lagos go unanswered. Response delays
kill people. There is no single coordination point connecting willing helpers
with people who need them in real time.

The solution: One WhatsApp number. Send a message describing what you see.
Within 90 seconds Claude AI verifies it, the nearest qualified community
responder gets GPS directions to the scene, the closest clinic is alerted,
and the reporter gets a live tracking link. No app download. No account.
No form. Just one WhatsApp message.

Beyond emergency response, Siren also:
- Lets anyone save locations they care about (home, child's school, land)
  and get WhatsApp alerts when incidents happen nearby
- Runs a live community resource board on each active incident where people
  can suggest and claim physical resources (cars, ladders, food, medical kits)
- Accepts donations for victims, responders, and the platform with Paystack
- Lets community members vouch for incidents to help verify them

--------------------------------------------------------------------------------
1.1 CORE PRINCIPLES — NEVER BREAK THESE
--------------------------------------------------------------------------------

1. WhatsApp is the entry point. Web is the richer experience.
   Use the "foot in the door" approach: get people started on WhatsApp
   with zero friction, then bring them to the website for richer actions
   (donating, managing subscriptions, viewing the full resource board,
   seeing the map). WhatsApp starts the relationship. The site deepens it.

2. Zero friction for reporters. One WhatsApp message. No account, no app,
   no form. If we add friction at the entry point, people don't report.

3. AI filters before humans see anything. Responders only see VERIFIED
   incidents. The feed stays trusted.

4. Every incident closes visibly. The reporter must see the loop close —
   what happened, who helped, how fast. This is what builds trust.

5. Government is optional. Platform works fully without LASEMA.

6. No account required for the public. Reporters, vouchers, resource
   contributors, and donors are identified by WhatsApp number or a
   short-lived session token. JWT login is only for agency operators
   and platform admin. These are two completely separate tracks.
   The landing-page-no-account approach does NOT conflict with
   authentication for the protected dashboard.

--------------------------------------------------------------------------------
1.2 THE SEVEN INCIDENT TYPES — ONLY THESE ARE ACCEPTED
--------------------------------------------------------------------------------

Code        What qualifies
----------- ----------------------------------------------------------------
FIRE        Structure fire, vehicle fire, gas explosion.
            NOT: trash fires, small cooking fires with no spread risk.
FLOOD       Homes or roads submerged, people trapped by rising water.
            NOT: puddles or mild waterlogging.
COLLAPSE    Full or partial building collapse, imminent collapse with people.
            NOT: cracks or cosmetic damage.
RTA         Road crash with injuries or trapped victims.
            NOT: minor fender benders with no injuries.
EXPLOSION   Gas tanker, pipeline, or industrial explosion with radius risk.
DROWNING    Person unable to self-rescue in water, boat capsize, flood rescue.
HAZARD      Live downed power lines or collapsed infrastructure threatening life.

Everything outside these seven is rejected at AI verification with a message
pointing the reporter to the right resource.

SECTION 6 — HAVERSINE DISTANCE UTILITY (utils/distance.py)
================================================================================

Create this file before writing any app. Every distance calculation in the
system uses this single function. It runs directly on Supabase's PostgreSQL.

IMPORTANT: The query uses HAVING (not WHERE) for the distance filter because
distance_km is a computed column alias. Do not change HAVING to WHERE.

--------------------------------------------------------------------------------

# utils/distance.py
from django.db import connection


def haversine_query(table, lat_field, lng_field, incident_lat, incident_lng,
                    radius_km, extra_filters='', extra_params=None, limit=10):
    """
    Returns rows from `table` within radius_km of (incident_lat, incident_lng).
    Ordered by distance ascending (nearest first).

    Args:
        table:          database table name e.g. 'responders_responder'
        lat_field:      latitude column name e.g. 'home_lat'
        lng_field:      longitude column name e.g. 'home_lng'
        incident_lat:   float latitude of the reference point
        incident_lng:   float longitude of the reference point
        radius_km:      maximum distance in kilometres
        extra_filters:  additional SQL e.g. "AND status = 'VERIFIED'"
        extra_params:   list of params for extra_filters placeholders
        limit:          max number of results

    Returns:
        List of dicts. Each dict has all table columns plus 'distance_km'.
    """
    base_params = [incident_lat, incident_lng, incident_lat]
    filter_params = extra_params or []
    params = base_params + filter_params + [radius_km]

    sql = f"""
        SELECT *,
          (6371 * acos(
            LEAST(1.0,
              cos(radians(%s)) * cos(radians({lat_field})) *
              cos(radians({lng_field}) - radians(%s)) +
              sin(radians(%s)) * sin(radians({lat_field}))
            )
          )) AS distance_km
        FROM {table}
        WHERE (1=1) {extra_filters}
        HAVING distance_km <= %s
        ORDER BY distance_km ASC
        LIMIT {limit}
    """

    with connection.cursor() as cursor:
        cursor.execute(sql, params)
        columns = [col[0] for col in cursor.description]
        return [dict(zip(columns, row)) for row in cursor.fetchall()]

import uuid
from django.db import models


class IncidentType(models.TextChoices):
    FIRE      = 'FIRE',      'Fire Outbreak'
    FLOOD     = 'FLOOD',     'Flooding'
    COLLAPSE  = 'COLLAPSE',  'Building/Structural Collapse'
    RTA       = 'RTA',       'Road Traffic Accident'
    EXPLOSION = 'EXPLOSION', 'Gas/Chemical Explosion'
    DROWNING  = 'DROWNING',  'Drowning/Water Emergency'
    HAZARD    = 'HAZARD',    'Structural/Electrical Hazard'


class IncidentSeverity(models.TextChoices):
    LOW      = 'LOW',      'Low'
    MEDIUM   = 'MEDIUM',   'Medium'
    HIGH     = 'HIGH',     'High'
    CRITICAL = 'CRITICAL', 'Critical'


class IncidentStatus(models.TextChoices):
    DETECTED        = 'DETECTED',        'Detected'
    VERIFYING       = 'VERIFYING',       'Verifying'
    VERIFIED        = 'VERIFIED',        'Verified'
    RESPONDING      = 'RESPONDING',      'Community Responding'
    AGENCY_NOTIFIED = 'AGENCY_NOTIFIED', 'Agency Notified'
    RESOLVED        = 'RESOLVED',        'Resolved'
    REJECTED        = 'REJECTED',        'Rejected'


class Incident(models.Model):
    id            = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    source        = models.CharField(max_length=20,
                      choices=[('WHATSAPP', 'WhatsApp'), ('WEB', 'Web Portal')])
    external_id   = models.CharField(max_length=200, blank=True)  # Twilio SID
    reporter_hash = models.CharField(max_length=64)               # SHA256 of phone
    reporter_phone = models.CharField(max_length=20, blank=True)  # Needed to send replies

    # Classification
    incident_type = models.CharField(max_length=20, choices=IncidentType.choices, blank=True)
    description   = models.TextField()
    severity      = models.CharField(max_length=10, choices=IncidentSeverity.choices,
                      default='MEDIUM')
    status        = models.CharField(max_length=20, choices=IncidentStatus.choices,
                      default='DETECTED')

    # Location — two plain FloatFields, NO GeoDjango
    location_lat  = models.FloatField(null=True, blank=True)
    location_lng  = models.FloatField(null=True, blank=True)
    address_text  = models.CharField(max_length=500, blank=True)
    zone_name     = models.CharField(max_length=100, blank=True)

    # Media
    media_urls    = models.JSONField(default=list)

    # AI verification
    ai_confidence   = models.FloatField(default=0.0)
    fraud_score     = models.FloatField(default=0.0)
    ai_raw_response = models.JSONField(default=dict)

    # Community verification
    vouch_count     = models.PositiveIntegerField(default=0)
    vouch_threshold = models.PositiveIntegerField(default=3)

    # Donations summary (updated whenever a donation is made)
    total_donations_kobo = models.PositiveIntegerField(default=0)  # Stored in kobo
    donation_count       = models.PositiveIntegerField(default=0)

    # Assigned agency
    agency_assigned = models.ForeignKey(
        'organisations.Organisation', null=True, blank=True,
        on_delete=models.SET_NULL, related_name='assigned_incidents'
    )

    created_at  = models.DateTimeField(auto_now_add=True)
    updated_at  = models.DateTimeField(auto_now=True)
    resolved_at = models.DateTimeField(null=True, blank=True)

    class Meta:
        ordering = ['-created_at']

    @property
    def vouch_threshold_for_severity(self):
        return {'CRITICAL': 1, 'HIGH': 2, 'MEDIUM': 3, 'LOW': 5}.get(self.severity, 3)

    def set_vouch_threshold(self):
        self.vouch_threshold = self.vouch_threshold_for_severity
        self.save(update_fields=['vouch_threshold'])

    @property
    def total_donations_naira(self):
        return self.total_donations_kobo / 100


class ResponseLog(models.Model):
    """Immutable record of every status change. Never delete these."""
    incident    = models.ForeignKey(Incident, on_delete=models.CASCADE,
                    related_name='response_logs')
    from_status = models.CharField(max_length=20, blank=True)
    to_status   = models.CharField(max_length=20)
    actor       = models.CharField(max_length=200)  # 'AI', 'community', 'agency', 'admin'
    note        = models.TextField(blank=True)
    created_at  = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ['created_at']


class VouchRecord(models.Model):
    """One vouch per session per incident. unique_together prevents duplicates."""
    incident      = models.ForeignKey(Incident, on_delete=models.CASCADE,
                      related_name='vouches')
    session_hash  = models.CharField(max_length=64)  # SHA256(IP + UserAgent + incident_id)
    voucher_lat   = models.FloatField(null=True, blank=True)
    voucher_lng   = models.FloatField(null=True, blank=True)
    distance_km   = models.FloatField(null=True, blank=True)
    is_suspicious = models.BooleanField(default=False)
    source        = models.CharField(max_length=20,
                      choices=[('WEB', 'Web'), ('WHATSAPP', 'WhatsApp reaction')],
                      default='WEB')
    created_at    = models.DateTimeField(auto_now_add=True)

    class Meta:
        unique_together = ('incident', 'session_hash')

import json, re
from celery import shared_task
from django.conf import settings
import anthropic


@shared_task(bind=True, max_retries=3, default_retry_delay=10)
def verify_incident_ai(self, incident_id: str):
    """
    Calls Claude to classify and verify an incident.
    Updates status. Triggers all downstream notifications if VERIFIED.
    """
    from apps.incidents.models import Incident

    incident = Incident.objects.get(id=incident_id)

    prompt = f"""You are an emergency verification system for Lagos, Nigeria.
Analyse this report and respond ONLY with valid JSON. No markdown. No explanation.

Report: {incident.description}
Location text: {incident.address_text}

Return this exact JSON:
{{
  "eligible": true or false,
  "rejection_reason": "why rejected, or null if eligible",
  "incident_type": "FIRE|FLOOD|COLLAPSE|RTA|EXPLOSION|DROWNING|HAZARD or null",
  "severity": "LOW|MEDIUM|HIGH|CRITICAL",
  "ai_confidence": 0.0 to 1.0,
  "fraud_score": 0.0 to 1.0,
  "suggested_skills": ["list of skill codes"],
  "reasoning": "one sentence"
}}

Only these types qualify: FIRE FLOOD COLLAPSE RTA EXPLOSION DROWNING HAZARD
fraud_score above 0.7 means the report looks fake."""

    try:
        client = anthropic.Anthropic(api_key=settings.ANTHROPIC_API_KEY)
        message = client.messages.create(
            model=settings.ANTHROPIC_MODEL,
            max_tokens=1024,
            messages=[{'role': 'user', 'content': prompt}]
        )
        text = message.content[0].text.strip()
        text = re.sub(r'```json|```', '', text).strip()  # Strip code fences
        result = json.loads(text)
    except Exception as exc:
        raise self.retry(exc=exc)

    incident.ai_raw_response = result
    incident.ai_confidence   = result.get('ai_confidence', 0.0)
    incident.fraud_score     = result.get('fraud_score', 0.0)

    if not result.get('eligible') or incident.fraud_score > 0.7:
        _transition(incident, 'REJECTED', 'AI', result.get('rejection_reason', ''))
        incident.save()
        _notify_rejected(incident, result.get('rejection_reason', ''))
        return

    incident.incident_type = result.get('incident_type', '')
    incident.severity      = result.get('severity', 'MEDIUM')
    incident.set_vouch_threshold()

    if incident.ai_confidence >= 0.75:
        _transition(incident, 'VERIFIED', 'AI',
                    f'Auto-verified. Confidence: {incident.ai_confidence:.2f}')
        incident.save()
        _post_verification_actions(incident)
    elif incident.ai_confidence >= 0.4:
        _transition(incident, 'VERIFYING', 'AI', 'Awaiting community vouches')
        incident.save()
        _notify_verifying(incident)
    else:
        _transition(incident, 'REJECTED', 'AI', 'Insufficient confidence')
        incident.save()


def _transition(incident, new_status, actor, note=''):
    from apps.incidents.models import ResponseLog
    from apps.incidents.consumers import broadcast_update
    ResponseLog.objects.create(
        incident=incident, from_status=incident.status,
        to_status=new_status, actor=actor, note=note
    )
    incident.status = new_status
    broadcast_update(incident)  # Push to all WebSocket clients


def _post_verification_actions(incident):
    from apps.whatsapp.tasks import notify_reporter_verified, post_community_announcement
    from apps.responders.tasks import notify_nearest_responders
    from apps.organisations.tasks import notify_nearest_organisations
    from apps.subscriptions.tasks import notify_location_subscribers

    notify_reporter_verified.delay(str(incident.id))
    post_community_announcement.delay(str(incident.id))
    if incident.severity in ['CRITICAL', 'HIGH', 'MEDIUM']:
        notify_nearest_responders.delay(str(incident.id))
    notify_nearest_organisations.delay(str(incident.id))
    notify_location_subscribers.delay(str(incident.id))

def route_inbound(from_number, body, media_urls, location):
    body_upper = body.strip().upper()

    # Responder commands
    if body_upper in ['YES', 'NO', 'ONSCENE', 'DONE', 'HELP']:
        return handle_responder_reply(from_number, body_upper)

    # Organisation commands
    if body_upper in ['ACCEPT', 'DECLINE', 'CALL']:
        return handle_org_reply(from_number, body_upper)

    # Registration
    if body_upper == 'RESPONDER':
        return start_responder_registration(from_number)
    if body_upper == 'REGISTER ORG':
        return start_org_registration(from_number)

    # Location subscriptions
    if body_upper == 'WATCH':
        return start_subscription_flow(from_number)
    if body_upper == 'MY ALERTS':
        return show_subscriptions(from_number)
    if body_upper.startswith('STOP'):
        return handle_stop_command(from_number, body)

    # Vouching
    if body_upper.startswith('VOUCH'):
        return handle_vouch_command(from_number, body)

    # Location share
    if location and location.get('latitude'):
        return handle_location_share(from_number, location, body)

    # Onboarding in progress
    if is_onboarding(from_number):
        return handle_onboarding_step(from_number, body)

    # Default: treat as emergency report
    return create_incident_from_message(from_number, body, media_urls, location)

--------------------------------------------------------------------------------
12.1 HOW PAYSTACK SPLITS WORK
--------------------------------------------------------------------------------

Siren uses Paystack sub-accounts to automatically split every donation
at transaction level. No manual money movement.

Setup (do once in Paystack dashboard):
1. Create Siren main account — this receives the platform percentage
2. Create sub-account for each verified responder when they join
3. Create a general "Victim Relief" holding sub-account

Split logic per fund_choice:
- PLATFORM: 100% to Siren main account
- RESPONDER: Siren takes 10%. Remaining 90% split equally among
  responders who have marked ONSCENE for this incident.
  If no responders on scene yet: hold in Siren account, redistribute
  when responders confirm on scene.
- VICTIM: Siren takes 10%. 90% to Victim Relief holding account.
  Released when reporter sets up recipient bank details.
  If no recipient set within 48 hours: donor notified, can redirect.

--------------------------------------------------------------------------------
12.2 PAYSTACK INTEGRATION FLOW
--------------------------------------------------------------------------------

Step 1 — Frontend initiates:
  POST /api/resources/donate/
  Body: {incident_id, amount_naira, fund_choice, donor_email, donor_phone}
  Backend creates Donation record (PENDING), generates paystack_reference,
  calls Paystack initialize API, returns {payment_url}

Step 2 — Frontend redirects donor to payment_url (Paystack hosted page)

Step 3 — Donor completes payment on Paystack

Step 4 — Paystack sends webhook to POST /api/resources/donate/verify/
  Backend validates Paystack signature (use X-Paystack-Signature header)
  Updates Donation.status to SUCCESS
  Updates Incident.total_donations_kobo
  Triggers responder payout split if fund_choice is RESPONDER
  Sends thank-you WhatsApp to donor if phone provided

Step 5 — Frontend callback URL /donate/success?ref={reference}
  Calls GET /api/resources/donations/?incident={id} to show updated total

@admin.register(Incident)

    list_display  = ['id', 'incident_type', 'severity', 'status', 'zone_name',
                     'vouch_count', 'ai_confidence', 'fraud_score',
                     'total_donations_kobo', 'created_at']
    list_filter   = ['status', 'incident_type', 'severity', 'source']
    search_fields = ['description', 'address_text', 'zone_name']
    ordering      = ['-created_at']
    readonly_fields = ['id', 'reporter_hash', 'ai_raw_response', 'created_at', 'updated_at']

    def mark_resolved(self, req, qs): qs.update(status='RESOLVED')

SECTION 16 — CRITICAL NOTES FOR CLAUDE CODE
================================================================================

Read all of these before writing a single line of code.

1.  NO GEODJANGO. EVER.
    Every models.py uses: from django.db import models
    NEVER: from django.contrib.gis.db import models
    NEVER: PointField, PolygonField, GEOSGeometry, or any gis import.
    All location data = two FloatFields: location_lat and location_lng.
    All spatial queries go through utils/distance.py using Haversine SQL.

2.  HAVERSINE USES HAVING NOT WHERE.
    The distance_km column is computed in the SELECT clause.
    The HAVING clause filters on it. Do not change HAVING to WHERE.
    This is correct PostgreSQL SQL. Do not "fix" it.

3.  TWILIO SIGNATURE VALIDATION IS MANDATORY.
    Every request to /api/ingest/whatsapp/ must be validated with:
    from twilio.request_validator import RequestValidator
    validator = RequestValidator(settings.TWILIO_AUTH_TOKEN)
    valid = validator.validate(url, POST_data, signature_header)
    Reject (return 403) if invalid.

4.  WHATSAPP NUMBER FORMAT.
    Twilio requires 'whatsapp:' prefix for all WhatsApp sends.
    Always format numbers as: 'whatsapp:+2348012345678'
    TWILIO_WHATSAPP_NUMBER in .env already has this prefix.
    The from_number in Twilio webhooks also has this prefix — strip it
    when storing: phone = from_number.replace('whatsapp:', '')

5.  CLAUDE RESPONSE CLEANING.
    Claude may wrap JSON in markdown code fences.
    Always clean before parsing:
    text = re.sub(r'```json|```', '', text).strip()
    Then: result = json.loads(text)

6.  PHONE PRIVACY.
    Store SHA256(phone_number) as phone_hash for privacy.
    Store the actual phone number in reporter_phone only where needed to
    send messages. Import hashlib. Use:
    hashlib.sha256(phone.encode()).hexdigest()

7.  JWT IS ONLY FOR AGENCY/ADMIN DASHBOARD.
    Public users (reporters, vouchers, donors, subscribers) never log in.
    They are identified by phone hash or session token.
    The no-account approach does not conflict with dashboard JWT auth.
    These are two separate authentication tracks.

8.  JWT STORAGE IN REACT.
    Store JWT tokens in React useState or Context only.
    NEVER localStorage or sessionStorage.
    Dashboard users log in fresh each session.

9.  LEAFLET CSS.
    import 'leaflet/dist/leaflet.css' must be the first import in main.tsx.
    Without it the map renders with no styles. This is not optional.

10. CELERY IN DEVELOPMENT.
    CELERY_ALWAYS_EAGER=True means tasks run synchronously in development.
    No Redis needed to test the AI verification flow locally.
    Set to False in production and ensure REDIS_URL is set.

11. SUPABASE CONNECTION STRING.
    Use the Direct Connection URI from Supabase Settings > Database.
    DO NOT use the pooler or transaction pooler string.
    The pooler does not support all query types used in this project.

12. PAYSTACK WEBHOOK VALIDATION.
    Validate the X-Paystack-Signature header on every donation webhook.
    Use HMAC SHA512 with PAYSTACK_SECRET_KEY as the key.
    Reject any request that fails validation.
    import hmac, hashlib
    signature = hmac.new(secret.encode(), body, hashlib.sha512).hexdigest()

13. DONATION AMOUNTS IN KOBO.
    Paystack uses kobo (100 kobo = 1 naira).
    Store all amounts in kobo in the database.
    Convert to naira only for display.
    amount_kobo = int(amount_naira * 100)

14. SUBSCRIPTION BROAD SWEEP.
    notify_location_subscribers uses a 10km broad sweep first,
    then filters by each subscription's own alert_radius_km.
    The 10km is intentional — do not change it to the individual radius.
    Individual radius filtering happens in the Python loop after the query.

15. RESOURCECLAIM DUPLICATE PREVENTION.
    unique_together = ('resource', 'claimer_hash') prevents one person
    claiming the same item twice. Always catch IntegrityError on claim creation
    and return a 200 (not 400) with message "You have already claimed this item".
